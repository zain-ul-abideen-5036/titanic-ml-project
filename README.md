## Overview

This project implements a production-grade machine learning system built on the Titanic survival dataset. It covers the complete engineering lifecycle — raw data ingestion, systematic feature engineering, model training and hyperparameter tuning, a validated REST API, a web dashboard, Docker containerization, automated testing via CI/CD, and live deployment on **Microsoft Azure Container Apps**.

The objective was not simply to train a model, but to build and ship it the way it would be done in a professional engineering environment: modular, tested, containerized, and deployed.

## Setup and Local Run

**1. Clone and create environment**

```bash
git clone https://github.com/zain-ul-abideen-5036/titanic-ml-project.git
cd titanic-ml-project

python -m venv venv
source venv/bin/activate        # Windows: venv\Scripts\activate
pip install -r requirements.txt
```

**2. Add the dataset**

Download `train.csv` from [kaggle.com/competitions/titanic/data](https://www.kaggle.com/competitions/titanic/data), rename it `titanic.csv`, and place it in `data/`.

**3. Train the model**

```bash
python src/train.py
```

Runs cleaning → feature engineering → GridSearchCV tuning → evaluation. Saves `artifacts/model.pkl` and `artifacts/scaler.pkl`.

**4. Start the server**

```bash
uvicorn api.main:app --reload
```

Open [http://localhost:8000](http://localhost:8000).

**5. Run tests**

```bash
pytest tests/ -v
```

## Machine Learning Details

### Feature Engineering

| Feature | Construction | Signal |
|---|---|---|
| `Title` | Extracted from `Name` field, grouped: Mr / Miss / Mrs / Master / Rare | `Master` (boys) had 57% survival vs 16% for `Mr` |
| `FamilySize` | `SibSp + Parch + 1` | Groups > 4 had lower survival rates |
| `IsAlone` | Binary: `FamilySize == 1` | Distinct survival pattern from any group |
| `Sex` | Encoded male → 0, female → 1 | Strongest predictor (importance: 0.281) |
| `Embarked` | One-hot encoded, first category dropped | Correlated with class and fare |
| `Age`, `Fare` | StandardScaler normalization | Scaler fitted on train data only, serialized separately to prevent data leakage |

### Model Comparison

| Model | Accuracy | F1 | ROC-AUC |
|---|---|---|---|
| Logistic Regression *(baseline)* | 79.3% | 76.8% | 84.2% |
| RandomForest *(default)* | 81.0% | 78.8% | 86.1% |
| **RandomForest + GridSearchCV** | **83.2%** | **80.7%** | **88.4%** |

## API Reference

### `POST /predict`

```json
// Request
{
  "pclass":   1,
  "sex":      "female",
  "age":      29.0,
  "sibsp":    0,
  "parch":    0,
  "fare":     100.0,
  "embarked": "S"
}

// Response
{
  "survived":    true,
  "probability": 0.9134,
  "confidence":  "high",
  "message":     "This passenger likely survived (probability: 91%)."
}
```

| Field | Values |
|---|---|
| `pclass` | `1`, `2`, `3` |
| `sex` | `"male"` or `"female"` |
| `embarked` | `"C"` (Cherbourg), `"Q"` (Queenstown), `"S"` (Southampton) |
| `confidence` | `"high"` ≥ 85% · `"medium"` 65–85% · `"low"` < 65% |

Invalid inputs return `HTTP 422` with field-level validation detail.

Interactive documentation available at `/docs`.

## CI/CD

GitHub Actions runs the full test suite on every push to `main`. The pipeline installs dependencies, executes all unit and integration tests, and reports pass/fail status on every commit.

Test coverage includes preprocessing transformations, feature construction, missing value handling, all four API endpoints, and input validation edge cases.
